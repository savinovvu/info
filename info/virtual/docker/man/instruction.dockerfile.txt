FROM – первая инструкция в скрипте, которая указывает, на основе какого базового образа будет собран текущий образ

RUN – здесь вы указываете команды (чаще всего операционной системы), которые будут выполняться при сборке образа

LABEL – позволяет добавить мета-данные в ваш образ (скорее всего информация об авторе либо какое-то примечание)
# LABEL Author="Sergey Morenets", Version=0.1


EXPOSE – позволяет указать порт(ы) в вашем контейнере, которые будут доступны извне, скорее всего для других контейнеров

ENV – здесь вы задаете переменные окружения в образе и если нужно указывает их начальные значения

ADD – копирует файлы или папки из вашей файловый системы в образ. Также поддерживает работу с архивами и удаленными файлами(Http)

COPY - копирует файлы или папки из вашей файловый системы в образ

ENTRYPOINT - указывает, какая команда будет запущена при старте контейнера. Может быть только одна

CMD – также указывает, какая команда будет запущена при старте контейнера. Здесь же можно указать параметры для ENTRYPOINT. Имеет несколько форм записи, как и предыдущая инструкция.

WORKDIR – указывает текущую папку в образе/контейнере при работе инструкций RUN/COPY/ADD/CMD/ENTRYPOINT. Вы можете указывать неограниченное количество таких инструкций в скрипте.
